I pulled Sega's official Disc Format Standards Specification Sheet (ST-040-R4-051795, via antime.kapsi.fi), and its System ID table is unambiguous: 0xE8 is STACK-M (the Master-SH2 stack pointer address), and the 1st Read Address — the transfer destination for the file loaded during the Sega logo — is at offset 0xF0. The load address value 0x06006000 is fine (your Ymir savestate fixture independently proved it), but the cited offset is wrong and should be 0xF0. Slightly ironic given the charter's own "always trust the disc's own header" line — worth checking whether tools/extract.py or sh2_map.py reads 0xF0 correctly and only the prose is wrong, or whether the wrong offset happens to work by coincidence of nearby values.
